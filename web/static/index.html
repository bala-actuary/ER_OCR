Pipeline

1. Split PDFs

Split multi-page downloads into single-page files

This splits each multi-page PDF into individual page files. All pages are split — non-data pages (metadata, summary, maps) are auto-detected and skipped during extraction in Step 3. Output goes to Input/split_files//{english,tamil}/.
2. Extract (OCR)

Run Tesseract OCR on each page pair

--part — Process specific part number or range (e.g., "101" or "50-100")
--page — Target specific pages within a part (e.g., "4", "1-10", "1,5,10-20"). Requires --part.
--limit — Max pairs to process (0 = all). Useful for testing a few pages first
--cross-check — Cross-validate EN vs TA cells, adds 2 extra columns (Cross_Check, Cross_Check_Notes)
--reset — Clear checkpoint for specified part, allowing re-processing
Dry Run — Show pending file pairs without processing
Validate Page — Test one page with detailed output (opens in Live Logs)
3. Merge Outputs

Combine page CSVs into part and AC-level files

4. Analyze Quality

Validate extraction accuracy and field fill rates
